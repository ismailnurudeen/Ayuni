# Ayuni Product QA

Use this skill when a ticket changes user-facing product behavior.

## Product Rules

- phone-first, low-friction signup
- only minimum profile required upfront
- 5 profiles per round
- next round at 8:00 PM WAT
- accepted history and declined history behave correctly
- no freeform chat before booking
- token payment required before booked date
- government ID required before first booked date
- support and safety states must be visible and actionable

## QA Questions

- does the flow feel intentional and compact?
- are we asking for too much too early?
- can the user recover from errors?
- is trust messaging clear without blocking too much?
- do ops and consumer views stay consistent?
